## Problem Statement

There is no directory of pubs serving DBK (Diabolo Banane-Kiwi — limonade + sirop de banane + sirop de kiwi, servi avec glaçons et une paille). The drink is a regional speciality, apparently common in Normandy but unknown in much of France. Florian wants to build a deliberately over-engineered "guide" to it as a portfolio / fun project — leaning into the absurdity of treating a fruit-syrup drink with Michelin-grade earnestness.

## What Makes This Cool

The product is **Le Guide DBK**: a Michelin-style *guide intérieur* for a single absurd drink. v1 is hand-curated reviews of 10 real Normandy pubs, written by a fictional persona ("Inspecteur DBK"), scored on a 1–5 star scale (Google-Maps style) across five sub-axes with deranged French rubric names ("Intensité du Diabolo," "Densité Glaçonnaire," "Équilibre Kiwi," "Audace de la Banane," "Verre & Service"). Real venues, satirical rigor — that tension is the whole product. v∞ becomes Untappd-for-DBK ("L'Index DBK") with user accounts and check-ins; v1 is the auteur-critic phase.

The "whoa" moment: someone in Normandy opens the site in a pub, sees their local scored 4.2/10 for "diabolo aqueux et sans tenue," and realizes someone wrote ten of these.

## Constraints

- **Stack:** Vite + vanilla JS/HTML/CSS. No framework. Node.js 24.
- **Data:** JSON files (`pubs.json`, `ratings.json`), keyed by stable `pub_id`.
- **Hosting:** Cloudflare Pages (free tier, custom domain, Git-auto-deploy, HTTPS by default).
- **Language:** **French only** in v1 UI chrome and reviews. Audience is French Twitter / r/france. Bilingual deferred to v1.1.
- **Privacy:** No cookies, no analytics, no third-party scripts. CNIL/GDPR consent banner not required in v1.
- **Offline:** Explicit non-goal in v1. No service worker. JSON files are tiny and Cloudflare's edge cache handles flaky 4G adequately. Service-worker / app-shell deferred to v1.1.
- **North star:** Mobile-first, used in the pub. Phone in noisy bar, big tap targets (≥44px), fast on flaky 4G.
- **Time budget:** ~48 hours of solo work (one weekend + one buffer day).

## Premises

1. **Satire IS the product, not a placeholder.** "Real venues, satirical rigor" is the load-bearing tension. Drop "real" → dies in 2 days. Drop "rigor" → meme not portfolio.
2. **v1 is a single-author guide, openly.** No fake review counts. The Michelin auteur voice ("Inspecteur DBK") is the conceit; pretending to aggregate fake votes would muddy it.
3. **Forward-compatible JSON shape.** Separate `pubs.json` and `ratings.json` keyed by `pub_id` so v2 (real OSM pubs) and v∞ (user check-ins) extend without rewriting the model.
4. **Mobile-first / in-pub usage is the v1 north star.** Twitter-shareability is delivered via **build-time OG images** per pub (one PNG per `/pub/:id` URL, set via `<meta property="og:image">`), so virality and in-pub UX don't fight and there's no runtime Canvas complexity.

## Approaches Considered

### Approach A: Le Pamphlet
Single static HTML page, all 10 reviews on one scroll. No router, no map, no OG-per-pub. Effort S (8h), Risk Low. Ships Saturday but doesn't show off architecture and skips the "près de moi" feature.

### Approach B: Le Guide *(chosen)*
Vite hash-routed app: `/` (ranked index) + `/pub/:id` (full reviews). Mobile-first stack layout. "Près de moi" tab using native Geolocation. Build-time OG images per pub URL. Effort M (1.5–2 weekends).

### Approach C: Le Guide Cinétique
Single scroll-driven story page with score-fill animations on each card. Distinctive but deep-link / SEO / 4G-jank tradeoffs make it a poor fit for the in-pub north star.

## Recommended Approach

**Approach B — Le Guide.** It resolves the in-pub vs Twitter-shareability tension via:
1. **Mobile-first pub-detail pages** for the in-pub use case.
2. **Build-time OG images** per `/pub/:id` URL — every pub link Twitter-renders as a beautiful card, no runtime Canvas, no iOS share-sheet pain, Lighthouse impact zero. Pipeline: `satori` (Vercel) renders JSX-as-JS-objects to SVG, then `@resvg/resvg-js` rasterises to PNG. If the OG build script errors, CI falls back to a single bundled `public/og-default.png` and emits a warning — deploy never blocks on OG generation.

Architecture extends cleanly to v2 (add MapLibre + Overpass) and v∞ (swap JSON for API).

### Router spec

Hash-based router (~30 lines), no `pushState`. Routes:

| Pattern         | View                  |
| --------------- | --------------------- |
| `#/`            | `index-view`          |
| `#/pub/:id`     | `pub-view` (validates `:id` against `pubs.json`; unknown → 404) |
| `#/nearby`      | `nearby-view`         |
| anything else   | `not-found-view`      |

No nested routes, no query params in v1.

### Architecture sketch

```
/

├── public/
│   └── data/
│       ├── pubs.json            # [{ pub_id, name, city, address, lat, lng, hours_text, ... }]
│       └── ratings.json         # [{ pub_id, sub_scores (1–5 each), blurb_md, author, date }]
├── src/
│   ├── main.js                  # Router init, view mounting
│   ├── views/
│   │   ├── index-view.js        # Ranked leaderboard, French copy
│   │   ├── pub-view.js          # /pub/:id full review
│   │   ├── nearby-view.js       # Geolocation-sorted list, with permission/error states
│   │   └── not-found-view.js    # Charming 404 in Inspecteur DBK voice
│   ├── components/
│   │   ├── pub-card.js          # Reusable score-forward card (typographic hero, no photo)
│   │   └── score-display.js     # Big star rating (mean, 1 decimal) + per-axis star breakdown
│   ├── lib/
│   │   ├── data.js              # Loads + joins pubs.json + ratings.json by pub_id
│   │   ├── geo.js               # Geolocation API + haversine distance
│   │   └── router.js            # Tiny hash-route handler
│   └── styles/
│       └── guide.css            # Self-hosted EB Garamond display + system body, French-paper accents
├── scripts/
│   └── build-og-images.js       # Satori-based, generates dist/og/<pub_id>.png at build
└── tests/
    ├── data.test.js             # Schema validation, pub_id uniqueness, rating-pub join completeness
    ├── geo.test.js              # Haversine math correctness, edge cases (poles, anti-meridian)
    └── og.test.js               # Asserts each pub_id has an OG png > 5KB; no pixel-diff snapshot
```

### Data schema (v1)

```jsonc
// pubs.json
{
  "pub_id": "le-coq-rouennais",      // slug, immutable, stable across versions
  "name": "Le Coq Rouennais",
  "city": "Rouen",
  "address": "12 rue Saint-Romain, 76000 Rouen",
  "lat": 49.4395,
  "lng": 1.0938,
  "hours_text": "Lun–Sam 16h–02h, Dim fermé", // free-text, no live "open now" logic in v1
  "status": "open"                    // "open" | "closed" (closed pubs hidden from index, but /pub/:id still resolves with a "Établissement fermé" banner so existing share links don't 404)
}

// ratings.json
{
  "pub_id": "le-coq-rouennais",
  "sub_scores": {                     // each 1–5 stars (integer), Google-Maps style
    "diabolo": 4,                      // Intensité du Diabolo
    "glacon": 3,                      // Densité Glaçonnaire
    "kiwi": 4,                        // Équilibre Kiwi
    "banane": 5,                      // Audace de la Banane
    "verre": 3                        // Verre & Service
  },
  "blurb_md": "Un diabolo d'une effervescence remarquable, frôlant la générosité indécente. Le kiwi…",
  "author": "Inspecteur DBK",
  "date": "2026-05-04"
}
```

Stars 1–5, Google-Maps style. The author sets the five 1–5 sub-scores; the overall rating is the **mean of the five**, displayed to one decimal with a partial star (e.g. 3.8★). The overall is computed (not stored). `data.test.js` validates each sub-score is an integer in 1–5 and tests the mean computation. (Changed 2026-05-25 from a /100 sum of 0–20 sub-scores.)

### Geolocation states (concrete)

| State                                         | Behavior                                                                                                        |
| --------------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| Permission granted, in Normandy               | List sorted by haversine distance, distance shown ("3.2 km").                                                   |
| Permission granted, outside Normandy          | List sorted by distance anyway. Banner: "Vous êtes loin de la Normandie — voici les 10 quand même."             |
| Permission denied                             | Static ranked list (default index order). Inline copy: "Active la géoloc pour voir les bars près de toi."       |
| Permission prompt timeout                     | Same fallback as denied. Wrapped via `Promise.race(getCurrentPosition, setTimeout(10s))` since iOS Safari's native timeout is unreliable. |
| HTTPS or user-gesture missing (browser quirk) | Same fallback as denied.                                                                                        |

**"In Normandy" detection**: bounding-box test on `lat ∈ [48.5, 50.1]` and `lng ∈ [-1.9, 1.8]`. Imprecise on the edges (catches part of Île-de-France, misses parts of Calvados coast); good enough for a banner. Département-precise checks deferred.

Manual city picker is a v1.1 feature, not v1.

### Typography & visual

- **Display**: EB Garamond 700, self-hosted via `@fontsource/eb-garamond` (full Latin subset, ~50KB woff2 — accepted; manual subsetting via `subfont` deferred to v1.1 unless Lighthouse fails). `<link rel="preload" as="font" crossorigin>` in `index.html`.
- **Body**: system stack (`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`) — fast, native feel.
- **Sizes**: body 16px / 1.55 line-height / max-width 65ch. Pub name (`<h1>`) 28px on mobile, 36px desktop. Star rating block (stars + decimal number) sized as the hero anchor: decimal number 96px on mobile, 128px desktop; stars scaled to match.
- **Tap targets**: ≥44×44px minimum (Apple HIG). List rows ≥56px tall.
- **Color**: bone (`#f5efe6`) bg, ink (`#1a1815`) text, single accent (`#7e2620` — wine-port red) for score and underlines. Contrast: ink-on-bone = 14.8:1 (AAA), accent-on-bone = 7.4:1 (AAA for normal text). No greens, no rainbows.
- **Critical CSS** inlined in `<head>`; rest deferred. Goal: First Contentful Paint < 1s on Slow 4G.
- **Accessibility**: focus order = DOM order, `<h1>` for pub name. Star rating exposed to screen readers as text, not just glyphs — e.g. `aria-label="Note globale : 3,8 sur 5"` and each axis as `<dl><dt>Intensité du Diabolo</dt><dd aria-label="4 sur 5">★★★★☆</dd></dl>`. Decorative star glyphs marked `aria-hidden`.
